# Strategy Translator Module

## Overview

The Strategy Translator acts as the bridge between the user-facing GUI and the core backtesting engine. It is responsible for converting the rules and settings defined in the Strategy Builder UI into executable C++ objects.

## Location

- **Source Code**: `src/strategy/`
- **Headers**: `src/strategy/`

## Responsibilities

- Parsing the strategy configuration generated by the GUI.
- Translating the UI-defined rules into a set of C++ condition objects that the backtesting engine can evaluate.
- Constructing and configuring technical indicator objects (e.g., SMA, EMA) required by the strategy.
- Assembling the final, executable strategy object to be passed to the backtesting engine.

## Components

- `/parser` - The `RuleParser` reads the configuration (e.g., from JSON) created by the GUI.
- `/translator` - The `RuleTranslator` converts the parsed rules into C++ `Condition` objects.
- `/indicators` - Contains the implementations for technical indicators like `SMA` (Simple Moving Average) and `EMA` (Exponential Moving Average).
- `/conditions` - Defines the `Condition` base class and concrete condition types that represent the logic of a trading rule.

## Threading

The translation process typically runs on the main UI thread when the user initiates a backtest. It produces a thread-safe strategy object that is then passed to the Backtesting Engine thread for simulation.
